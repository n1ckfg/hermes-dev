# Serve Website Skill

This skill ensures a clean web server is running on port 8080 serving the contents of `~/hermes-workspace` with directory listing enabled.

## When to Use
- You need a quick HTTP server to test files in the workspace.
- You want to expose the workspace over Tailscale securely.
- Before running browser validation or visual tests.

## Prerequisites
- Node.js with `serve` package installed globally (`npm install -g serve`), **or** Python 3 (fallback).
- Tailscale running (optional, for safety note).

## Steps

### 1. Kill Existing Process on Port 8080
```bash
# Find and kill any process listening on TCP 8080
lsof -ti:8080 | xargs kill -9 2>/dev/null || true
# Alternative using fuser
fuser -k 8080/tcp 2>/dev/null || true
```

### 2. Launch Web Server with Directory Listing
Prefer `serve` (Node.js) if available; otherwise use Python's http.server.

#### Option A: Using `serve`
```bash
cd ~/hermes-workspace
npx serve . -l 8080
```
- `serve` automatically provides directory listing.

#### Option B: Using Python (fallback)
```bash
cd ~/hermes-workspace
python3 -m http.server 8080 --bind 0.0.0.0 --directory .
```
*Note: Python's http.server does **not** enable fancy directory listing by default; it shows a simple list. If you prefer a nicer listing, install `serve`.*

### 3. Verify Server is Running
```bash
# Should show something listening on 0.0.0.0:8080
ss -ltnp | grep 8080
# Or curl
curl -s http://localhost:8080/ | head -5
```

### 4. Safety Note (Tailscale)
- This server binds to `0.0.0.0` (all interfaces).  
- If you only want it reachable via Tailscale, ensure your firewall blocks external access or rely on Tailscale's ACLs.  
- For a quick test, you can expose via `tailscale serve` but that's outside this skill's scope.

## Cleanup
To stop the server when done:
```bash
lsof -ti:8080 | xargs kill -9 2>/dev/null || true
```

## Troubleshooting
- **Address already in use**: Ensure step 1 succeeded; maybe another service (like a previous `serve`) is still alive.
- **Permission denied**: You may need `sudo` to kill processes not owned by you; prefer running as the same user.
- **No output**: Check server logs; for `serve` you'll see output in terminal; for Python, it logs to stdout.

## Example Usage in a Workflow
1. Run this skill to start the server.
2. Use browser tools (`browser_navigate`, `browser_console`) to inspect pages.
3. Run validation scripts.
4. When finished, manually kill the server or re-run step 1.
